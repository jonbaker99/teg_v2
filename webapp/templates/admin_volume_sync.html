Selectively move files between GitHub and the app's data store ({{ store_label }}). Pull copies GitHub → store; Push copies store → GitHub. Use this when you change reference CSVs for new TEGs and want to sync just those files.

{# Environment banner — the "store" means very different things on Railway vs local. #} {% if is_railway %}
🟢 Live (Railway). The store is the persistent volume the live site reads from.
Pull refreshes the volume from GitHub and the site updates immediately (no redeploy); it does not commit. Push commits the volume's copy to GitHub. Pulls back up the volume file first (restore below).
{% else %}
⚠️ Local dev — the store is your git working tree, not a volume.
Pull overwrites your working-tree data/* files with GitHub's versions (shows up in git status; a backup is taken first — see Restore below). Push makes an out-of-band API commit to your current branch, so your local clone ends up behind origin until you git fetch. Use here for testing the flow; prefer normal git for real data work.
