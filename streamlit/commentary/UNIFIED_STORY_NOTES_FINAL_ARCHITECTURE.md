# Unified Story Notes - Final Architecture

**Status:** ✅ **IMPLEMENTED**
**Date:** 2025-01-15

---

## Overview

The unified story notes system uses a **single comprehensive data loader** with **individual round story notes files** that can be generated independently or as part of tournament commentary.

---

## Architecture

### **Data Flow:**

```
┌─────────────────────────────────────────────────────────────┐
│                UNIFIED ROUND DATA LOADER                     │
│         (unified_round_data_loader.py)                      │
│                                                              │
│  Loads ALL data for a round:                               │
│  • Hole-by-hole scoring                                    │
│  • Momentum patterns (hot/cold spells)                     │
│  • Front 9/Back 9 analysis                                 │
│  • Position tracking (dual competition)                    │
│  • Course context & records                                │
│  • Location context & venue history                        │
│  • 17 comprehensive data sections                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│           INDIVIDUAL ROUND STORY NOTES FILES                │
│                                                              │
│  Path: data/commentary/round_reports/TEG{X}_R{Y}_story_notes.md
│                                                              │
│  Generated by EITHER:                                       │
│  1. Tournament commentary (all rounds at once)             │
│  2. Round report generation (live, one round at a time)    │
│                                                              │
│  Smart regeneration:                                        │
│  • Check if exists before generating                       │
│  • Skip if exists (unless --force flag)                    │
│  • No duplicate LLM calls                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
                    ┌───────┴────────┐
                    │                 │
            ┌───────▼──────┐  ┌─────▼──────────┐
            │  TOURNAMENT   │  │  ROUND REPORT  │
            │  SYNTHESIS    │  │  GENERATION    │
            │               │  │                │
            │ Reads all 4   │  │ Reads 1 round │
            │ round notes   │  │ story notes   │
            │ files         │  │ file          │
            └───────────────┘  └────────────────┘
```

## Key Files

### **1. Data Loading**
- **`unified_round_data_loader.py`** (NEW - 942 lines)
  - Single comprehensive data loader
  - 17 data sections per round
  - Used by both tournament and round report systems

### **2. Story Notes Generation**

**Tournament Commentary:**
- **`generate_tournament_commentary_v2.py`**
  - `generate_round_story()` - Generates/loads individual round story notes
  - Checks if story notes exist before generating
  - Saves to `data/commentary/round_reports/TEG{X}_R{Y}_story_notes.md`
  - Tournament synthesis reads all individual files

**Round Reports:**
- **`generate_round_report.py`**
  - `generate_round_story_notes()` - Generates/loads individual round story notes
  - Uses same unified data loader
  - Checks if story notes exist before generating
  - Saves to same location as tournament system

### **3. Prompts**
- **`prompts.py`**
  - `ROUND_STORY_PROMPT` - Updated with all unified data sections
  - `ROUND_REPORT_PROMPT` - Updated for hot/cold spells

## Benefits

### ✅ **Single Source of Truth**
- One story notes file per round
- No conflicts between tournament and round reports
- Both systems use identical factual foundation

### ✅ **Cost Savings**
- ~50% reduction in LLM costs
- No duplicate story notes generation
- Smart caching: only generate if missing

### ✅ **Live Tournament Support**
- Generate round reports during tournaments
- Don't need to wait for tournament to complete
- Each round is independent

### ✅ **Flexibility**
- Generate rounds individually or all at once
- Regenerate specific rounds with `--force`
- Tournament synthesis assembles from individual files

### ✅ **Richer Data**
- All reports have access to unified data:
  - Hot/cold spells (3-hole and 6-hole windows)
  - Front 9/Back 9 patterns (not 6/6/6)
  - Course records (before/after tracking)
  - Location context (area returns)
  - Dual competition tracking (Trophy + Jacket)

## Usage Examples

### **Generate Round Report (Live Tournament)**
```bash
# During a tournament - generates story notes for this round only
python streamlit/commentary/generate_round_report.py --teg 18 --round 3

# Story notes saved to: data/commentary/round_reports/TEG18_R3_story_notes.md
```

### **Generate Tournament Commentary (Complete Tournament)**
```bash
# After tournament completes - generates all rounds
python streamlit/commentary/generate_tournament_commentary_v2.py 18

# Story notes saved to:
# - data/commentary/round_reports/TEG18_R1_story_notes.md
# - data/commentary/round_reports/TEG18_R2_story_notes.md
# - data/commentary/round_reports/TEG18_R3_story_notes.md
# - data/commentary/round_reports/TEG18_R4_story_notes.md

# Tournament synthesis reads all 4 files
```

### **Regenerate Specific Round**
```bash
# Force regeneration of Round 2 story notes
python streamlit/commentary/generate_round_report.py --teg 18 --round 2 --force
```

## Data Specifications

### **Round Story Notes Include:**

1. **Metadata** (8 fields)
   - TEG, round, course, date, area, par, rounds remaining

2. **Round Summary** (18 fields per player)
   - Stableford & Gross scores, positions, gaps
   - Front 9/Back 9 performance

3. **Position Tracking** (dual competition)
   - Position after each hole (Trophy & Jacket)
   - Movement from start of round

4. **Key Moments** (events)
   - Eagles, disasters, lead changes
   - With hole numbers, scores, and impact

5. **Hot/Cold Spells**
   - 3-hole and 6-hole momentum windows
   - Both Net (Stableford) and Gross
   - Specific hole ranges and details

6. **Front 9 vs Back 9 Patterns**
   - Significant differences (≥5 points Stableford, ≥4 gross)
   - Strong starters vs finishers

7. **Course Context** (NEW)
   - Course history (previous TEGs)
   - Course records: before/after for score, vs par, Stableford
   - New record detection

8. **Location Context** (NEW)
   - Area and year
   - Return status (first time, consecutive, gap)
   - Previous TEGs in area

9. **Tournament Projections**
   - Rounds remaining
   - What's needed to catch leader (both competitions)
   - Mathematical possibilities

10. **Records & Personal Bests**
    - All-time TEG records
    - Personal bests and worsts

**Total: 1200-2000 data points per round**

## Removed

### ❌ **Six-Hole Splits (1-6, 7-12, 13-18)**
- Replaced by Front 9/Back 9 (more natural)
- Hot/cold spells cover detailed momentum tracking

### ❌ **Combined Tournament Story Notes File**
- No longer needed
- Individual round files are assembled by synthesis
